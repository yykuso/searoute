# おでかけ航路マップ 仕様書

## 1. 文書の目的

この文書は、おでかけ航路マップの現行仕様を Spec-Driven Development 風に整理し、機能改修時の影響確認と回帰テストの基準として使うための設計書である。

主な用途は以下の通り。

- 仕様の基準線を明文化する
- 改修時に確認すべき振る舞いを明確にする
- 画面、データ、状態管理、外部依存の関係を把握しやすくする
- 手動テスト観点を漏れにくくする

## 2. 対象範囲

対象は GitHub Pages 上で配信される静的 Web アプリケーション全体。

- マップ画面: index.html
- 航路一覧画面: routeList.html
- PWA 関連: manifest.json, service-worker.js
- データ表示ロジック: js 配下のモジュール群
- 航路・港湾データ: data 配下の GeoJSON / JSON

対象外。

- 航路データ作成ツールの詳細仕様
- データ入力ワークフロー
- サーバーサイド処理

## 3. システム概要

本サイトは、日本国内外の旅客船・フェリー航路情報を地図上で探索できる静的 Web アプリである。利用者は、地図上の航路や港湾をクリックして詳細情報を確認し、外部の時刻表や公式サイトへ遷移できる。

主要な利用シナリオは以下。

- 地図上で航路を探す
- 航路の事業者名、航路名、運行頻度、備考、船名を確認する
- 航路の公式サイトや時刻表を開く
- 港湾位置を地図上で確認する
- 一覧ページで航路を検索する
- スマートフォンで PWA として利用する

## 5. データモデル

### 5.1 航路データ

航路データは GeoJSON と詳細 JSON の組み合わせで構成される。

- GeoJSON: 線形データと最小限の属性を保持
- Details JSON: routeId をキーとして詳細情報を保持

主なデータセット。

- data/seaRoute.geojson + data/seaRouteDetails.json
- data/internationalSeaRoute.geojson + data/internationalSeaRouteDetails.json
- data/seaRouteKR.geojson + data/seaRouteKRDetails.json
- data/limitedSeaRoute.geojson + data/limitedSeaRouteDetails.json

主なプロパティ。

- routeId: 航路識別子
- lineId: 航路区間識別子
- businessName: 事業者名
- routeName: 航路名
- portName1, portName2: 区間の両端港名
- color: 航路表示色
- freq: 線幅計算用の頻度指標
- freqInfo: 運行頻度表示文言
- info: 補足情報
- shipName: 船名一覧
- url: 時刻表または公式サイト URL
- note: 状態属性

### 5.2 note の意味

- null または未設定: 通常航路
- season: 季節運航
- suspend: 運休・休止中

この note 値は表示スタイル、一覧の行スタイル、表示フィルターに影響する。

### 5.3 港湾データ

港湾データは data/portData.geojson を使用する。主なプロパティは港名と補足情報である。

## 6. アーキテクチャ概要

### 6.1 技術要素

- 配信: GitHub Pages
- 実装方式: Vanilla JavaScript モジュール
- 地図: MapLibre GL JS
- 検索補助: Maplibre Geocoder + Nominatim
- スタイル: Tailwind CSS と独自 CSS
- PWA: manifest.json と service-worker.js
- 計測: Google Analytics

### 6.2 モジュール責務

- js/common.js: 地図初期化、ベースマップ切替、レイヤー状態管理、コンテキストイベント
- js/geoJsonLayers.js: GeoJSON レイヤー追加、クリックイベント、詳細表示、航路ズーム、休止中表示切替
- js/dataLoader.js: GeoJSON / 詳細 JSON の取得、結合、ローディング制御
- js/detailDrawer.js: 詳細ドロワーの表示制御、モバイルスナップ動作
- js/layersControl.js: レイヤー切替 UI の生成とイベント配線
- js/layerConfig.js: レイヤー構成定義
- js/routeList.js: 一覧テーブル生成、検索、リセット
- js/cookieControl.js: Cookie の読み書き

## 7. 機能仕様

### 7.1 MAP-01 初期表示

目的。

利用者がページを開いた直後に、保存済み状態または既定状態で地図を表示する。

入力。

- Cookie の currentMap
- Cookie の currentLayer
- Cookie の mapCenter
- Cookie の mapZoom

期待動作。

- Cookie が存在する場合は保存済みの地図種別、表示レイヤー、中心座標、ズームを復元する
- Cookie が存在しない場合は既定状態で初期表示する
- 既定のデータレイヤーは日本国内航路である
- 地図ロード完了後に詳細ドロワー領域を表示可能にする

受け入れ条件。

- 画面リロード後も前回の中心座標とズームが維持される
- 表示中レイヤーが再現される
- ベースマップが再現される

### 7.2 MAP-02 ベースマップ切替

目的。

利用者が背景地図を切り替えられるようにする。

対象種別。

- OSM Custom
- OSM Bright
- OSM Planet
- 地理院 標準
- 地理院 淡色
- 地理院 白地図
- OpenTopoMap
- TransportMap

期待動作。

- ベースマップ変更時に既存の GeoJSON / Tile オーバーレイ状態を維持する
- 必要に応じて raster レイヤーを削除・再追加する
- currentMap を Cookie に保存する

受け入れ条件。

- 切替後もデータレイヤーが消えない
- 港マーカー画像が必要時に再登録される
- 再読み込み後に切替結果が保持される

### 7.3 MAP-03 オーバーレイ / データレイヤー切替

目的。

利用者が raster オーバーレイと GeoJSON データレイヤーを個別に表示・非表示できるようにする。

対象オーバーレイ。

- tile_gsi_photo
- tile_gsi_relief
- tile_esriimagery
- tile_railwaymap
- tile_openseamap

対象 GeoJSON レイヤー。

- geojson_port
- geojson_sea_route
- geojson_international_sea_route
- geojson_KR_sea_route
- geojson_limited_sea_route

期待動作。

- レイヤー追加時にソースとレイヤーを正しく登録する
- レイヤー削除時に関連レイヤーとソースを削除する
- レイヤー順序は layerPriorities に従う
- currentLayer を Cookie に保存する

受け入れ条件。

- チェックボックス操作で表示状態が切り替わる
- 同一レイヤー追加時に二重登録しない
- 非表示後に再表示しても動作が壊れない

### 7.4 MAP-04 航路描画

目的。

航路種別ごとに線・ラベルを描画し、属性に応じた表示差分をつける。

期待動作。

- 各航路レイヤーは outline, solidline, dashline, thinline, name で構成される
- note に応じて通常線、季節線、休止中線を描き分ける
- color があれば線色と文字色に反映する
- freq に応じてズーム別に線幅を変化させる
- ラベルはズームに応じて事業者名、航路名の表示量を変える

受け入れ条件。

- season は破線、suspend は細い点線として表示される
- ラベルが線上に追従する
- 航路種別ごとにデータソースが正しい

### 7.5 MAP-05 港湾描画

目的。

港湾位置をマーカーと港名で表示する。

期待動作。

- anchor_marker 画像を登録する
- 港湾データソースを読み込む
- シンボルレイヤーとしてアイコンと名称を表示する

受け入れ条件。

- 港湾レイヤーを ON にするとアイコンと名称が表示される
- スタイル変更後も再表示可能である

### 7.6 MAP-06 航路クリック詳細

目的。

利用者が航路をクリックした際に、詳細ドロワーで情報を確認できるようにする。

表示内容。

- 事業者名
- 事業者名の補助表記
- 航路名
- 選択区間名
- 運行頻度
- 補足情報
- 船名一覧
- 外部リンク

期待動作。

- クリック対象の properties を詳細ドロワーに反映する
- 事業者名に全角括弧が含まれる場合は主表記と副表記に分離する
- 航路名または区間名クリックでズーム関数を呼ぶ
- marker_click イベントを計測する

受け入れ条件。

- 情報が欠損している項目はセクションを表示しない
- URL が不正でも画面生成自体は失敗しない
- HTML エスケープにより表示文字列が壊れにくい

### 7.7 MAP-07 港湾クリック詳細

目的。

利用者が港をクリックした際に、港湾情報を詳細ドロワーで確認できるようにする。

表示内容。

- 港名
- Google マップ検索リンク
- 補足情報

期待動作。

- 港名に 港 または 桟橋 を含まない場合は検索語に 港 を補う
- 詳細ドロワーに Google マップへのリンクを表示する

受け入れ条件。

- 港名クリック後にドロワーが開く
- 情報がない場合でも最低限の地図リンクは表示される

### 7.8 MAP-08 航路ズームとハイライト

目的。

利用者が航路全体または区間を地図上で見つけやすいようにする。

期待動作。

- routeId または routeId + lineId から対象 feature を取得する
- source キャッシュがなければ map ソースからフォールバック取得する
- feature 群から境界ボックスを計算する
- ドロワー表示状態に応じて fitBounds の padding を調整する
- 対象 feature を route-highlight として強調表示する
- 別地点クリックまたはドロワー閉鎖でハイライトを解除する

受け入れ条件。

- 航路名クリックで航路全体へズームする
- 区間名クリックで区間または routeId フォールバック対象へズームする
- ハイライトが二重に残らない

### 7.9 MAP-09 休止中航路の表示設定

目的。

利用者が休止中航路の表示有無を切り替えられるようにする。

期待動作。

- 設定ウィンドウのトグルで切り替える
- Cookie showSuspendedRoutes に保存する
- OFF 時は thinline を非表示にし、outline と name は suspend を除外する
- ON 時は元のフィルターを復元する

受け入れ条件。

- トグル変更後に即座に表示が変わる
- 再読み込み後も設定が維持される

### 7.10 MAP-10 地名検索

目的。

利用者が地名検索で地図位置を移動できるようにする。

期待動作。

- Nominatim の forward geocode を利用する
- 候補結果から中心座標を算出する
- Geocoder UI を top-left に表示する

受け入れ条件。

- 一般的な地名で候補が返る
- API 失敗時に画面全体が壊れない

### 7.11 MAP-11 コンテキストメニューと長押し座標取得

目的。

PC では右クリック、モバイルでは長押しで座標確認と外部地図導線を提供する。

期待動作。

- PC では contextmenu でカスタムメニューを表示する
- モバイルでは長押しでピンを置き、詳細ドロワーを開く
- 長押し時に逆ジオコーディングで住所を取得する
- ドロワーを閉じたらピンを削除する

受け入れ条件。

- 長押し中にドラッグした場合は発火しない
- 住所取得失敗時は失敗メッセージを表示する

### 7.12 MAP-12 詳細ドロワー

目的。

PC とモバイルで詳細情報を見やすく提示する。

期待動作。

- PC では左サイドから表示する
- モバイルでは 30%、80%、最小高さのスナップ挙動を持つ
- ドラッグ、外側タップ、閉じるボタンに対応する
- ドロワーを閉じるとロングプレスピンと航路ハイライトも解除する

受け入れ条件。

- モバイルでスワイプに応じて高さが変わる
- リサイズ時に表示破綻しない
- 閉鎖時に残留状態が消える

## 8. 状態管理

Cookie を利用して以下を保持する。

- currentMap: ベースマップ ID
- currentLayer: 表示中レイヤー ID のカンマ区切り
- mapCenter: 地図中心座標
- mapZoom: 地図ズーム値
- showSuspendedRoutes: 休止中航路表示フラグ

保持期間。

- currentMap, currentLayer, mapCenter, mapZoom: 30 日
- showSuspendedRoutes: 365 日

## 9. 外部依存

外部依存が失敗した場合でも、可能な範囲で画面全体の利用不能を避けることを基本方針とする。

主要依存。

- MapLibre GL JS
- Maplibre Geocoder
- Nominatim 検索 API
- Google Maps URL スキーム
- Google Analytics
- 各種タイルサーバー

注意点。

- Nominatim 失敗時は検索結果や住所取得が空になる可能性がある
- タイルサーバー障害時は背景地図のみ不完全になる可能性がある
- Analytics 失敗は主要機能に影響してはならない

## 10. 非機能要件

### 10.1 可用性

- 静的ファイルのみで配信可能であること
- 外部 API の一部失敗時でも主要画面を継続利用できること

### 10.2 操作性

- PC とモバイルの双方で主要機能が使えること
- モバイルでは長押し、スワイプ、PWA 表示に対応すること

### 10.3 保守性

- レイヤー追加時は layerConfig と対応データ、描画定義の整合が取りやすいこと
- 仕様変更時に回帰テスト観点が追跡可能であること

5. 航路クリックで詳細ドロワーが開く
6. 港クリックで詳細ドロワーが開く
7. 航路詳細の航路リンクで該当範囲へズームする

### 12.3 モバイルテスト

### 12.5 PWA / キャッシュテスト

2. service worker が登録される
3. 主要静的ファイルがキャッシュされる

## 14. 今後の運用方針

機能追加・改修時は、最低でも以下をセットで更新する。

1. 本仕様書の該当章
2. 影響した受け入れ条件
3. 回帰テスト項目
4. 必要なら README の利用者向け説明
